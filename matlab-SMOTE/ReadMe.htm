ReadMe file for CSNN Package

Description: This package contains 6 algorithms for training cost-sensitive neural networks. They are over-sampling, under-sampling, threshold-moving, SMOTE and two ensemble methods, i.e. hard-ensemble and soft-ensemble.

Reference:   Z.-H. Zhou and X.-Y. Liu. Training cost-sensitive neural networks with methods addressing class imbalance problem. IEEE Transactions on Knowledge and Data Engineering, 2006, 18(1): 63-77.

ATTN:         This package is free for academic usage. You can run it at your own risk. For other purposes, please contact Prof. Zhi-Hua Zhou (zhouzh@nju.edu.cn ).

Requirement: To use this package, Matlab 6.0 above with the neural network toolbox is required.

ATTN2:        This package was developed by Ms. Xu-Ying Liu ( liuxy@lamda.nju.edu.cn ). There is a ReadMe file provided for roughly explaining the codes. For any problem concerning the code, please feel free to contact with Ms. Liu.

Data format:

1.        Instance matrix: row indexes instances and column indexes attributes.

2.        Class label vector for instances: row vector.

3.        Class type: the i-th entry is the name of i-th class. With this data type, class labels can be freely encoded. Otherwise, class labels must be 1,2,….N when there are N classes.

4.        Cost matrix: cost[i,j] denotes the cost of misclassifying a j-th class instance as belonging to i-th class;

5.        Cost vector: cost vector. The i-th entry is the cost of misclassifying the i-th class instance, without considering the concrete class the instance has been wrongly assigned to.

6.        0-1 class label vector: each class corresponds to a 2-value (0 or 1) attribute, then a class label be can converted into a vector where value is 1 when the instance is belonging to the corresponding class and 0 otherwise. For example, there's 3 classes: 1 2 3, a class 2’s instance should be converted to [0 1 0].

7.        Real-value output of neural networks: taking 0-1 class label vector as its class label format. Output is a matrix when there are more that one instances and the row indexes class type while the column indexes instances.

8.        Attribute information

a)        Attribute vector: For attributes of a data set, 1 presents for the corresponding attribute is nominal and 0 for numeric.

b)        Attribute structure for pairwise VDM [5] distance of values on nominal attribute:

                         i.              FIELD kind: ‘nominal’ or ‘numeric’;

                       ii.              FIELD values: values on the nominal attribute or [] for numeric attribute

                      iii.              FIELD VDM: VDM[i,j] denotes VDM distance between the i-th value and the j-th value on the nominal attribute or VDM is [] for numeric attribute

Files contained in this package are listed below. For more information of the functionality and the usage of these algorithms, please refer to the paper and the corresponding codes and comments.

1.        ThresholdMovNN.m

Implement threshold-moving algorithm.

It is a post process of Neural Network to make NN cost sensitive by threshold moving toward inexpensive class, therefore it becomes harder for higher-cost instances to be misclassified.

To use this method:

step 1: train a cost-blind neural network N

step 2: test exampling using N to get real-value outputs o

step 3: using FUNCTION ThresholdMovNN to get cost-sensitive prediction or its real-value output o*

There’s also a sample routine named sample_ThresholdMovNN.m illustrating how to use this function.

2.        OverSampling.m

Implement over-sampling algorithm.

It changes the training data distribution by duplicating higher-cost training examples until the appearances of different training examples are proportional to their costs. Here, the duplication method is random over-sampling with replacement.

To use this method:

step 1: Generate new training set using FUNCTION OverSampling

step 2: Train a neural network N from the new training set

step 3: Get cost-sensitive predictions or their real-value outputs using N

There’s also a sample routine named sample_OverSampling.m illustrating how to use this function.

3.        UnderSampling.m

Implement under-sampling algorithm.

It changes the training data distribution by deleting some lower-cost training examples until the appearances of different training examples are proportional to their costs. Here a routine similar to that used in [2] is employed, which removes redundant examples at first and then removes borderline examples, the latter can be detected by Tomek links [3].

To use this method:

step 1: Generate new training set using FUNCTION UnderSampling

step 2: Train a neural network N from the new training set

step 3: Get cost-sensitive predictions or their real-value outputs using N

There’s also a sample routine named sample_UnderSampling.m illustrating how to use this function.

4.        SMOTE.m

Implement SMOTE algorithm [4].

This algorithm resamples the small class through taking each small class example and introducing synthetic examples along the line segments joining its small class nearest neighbors.

5.        SmoteOverSampling.m

Implement Smote algorithm.

Like over-sampling method, it changes the training data distribution by duplicating higher-cost training examples until the appearances of different training examples are proportional to their costs. Here, the duplication method is SMOTE.

To use this method:

step 1: Generate new training set using FUNCTION SmoteOverSampling

step 2: Train a neural network N from the new training set

step 3: Get cost-sensitive predictions or their real-value outputs using N

There’s also a sample routine named sample_SmoteOverSampling.m illustrating how to use this function.

6.        HardEnsemble.m

Implement hard-ensemble method.

In this method, every component learner votes (0-1 vote method) for a class and then the class receiving the biggest number of votes is returned. If a tie appears, that is, there are multiple classes receiving the biggest number of votes, then the class with the biggest cost is returned.

There’s also a sample routine named sample_HardEnsemble_SoftEnsemble.m illustrating how to use this function.

7.        SoftEnsemble.m

Implement soft-ensemble method.

In this method, every component learner votes using real-values for a class and then the class receiving the biggest number of votes is returned. If a tie appears, that is, there are multiple classes receiving the biggest number of votes, then the class with the biggest cost is returned.

There’s also a sample routine named sample_HardEnsemble_SoftEnsemble.m illustrating how to use this function.

8.        Utilities:

A.       CostMatrix.m

Generate 3 types of cost matrix defined in [1].

Cost matrix format: 1) cost[i,j] denotes the cost of misclassifying a j-th calss instance as belonging to i-th class;

2) integer values;

3) cost is zero if the example is classified correctly;

4) at least one cost value is 1.

Type a: 1.0<cost[i,j]<=max_value for a single i, cost[k≠i,j]=1.0 for all k≠j

Type b: 1.0<=cost[i,j]<=max_value and for each j≠i1 and j≠i2, cost[i1,j]=cost[i2,j]

Type c: 1.0<=cost[i,j]<=max_value for all i≠j


B.       normalize.m

Normalize column vector in a matrix

C.       LabelFormatConvertion.m

 Conversion between scalar class label format and 0-1 class label vector format.  The latter is required when training a neural network with real-value outputs.

D.      Locate.m

Locate a value in a set.

E.       VDM.m

Calculate VDM [5] for nominal attributes.

F.        dist_normimal.m

 Calculate distance where the distance of values on nominal attributes is VDM distance [5].
